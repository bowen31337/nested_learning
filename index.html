The "Deep Learning" View

Traditionally, we see models as stacks of layers. Information passes forward, gradients pass backward.

The Problem: This view hides the internal gradient flows. It creates an "illusion" of depth where layers are static and uniform, updated only by a single centralized clock (the global optimizer).

The "Nested Learning" View

The paper proposes viewing the model as a set of nested optimization problems.

  • speed
    Multi-Time Scale: Just like brain waves, different parts update at different frequencies (fast adaption vs. slow consolidation).
  • account_tree
    Context Flow: Each nesting level (Inner loop vs Outer loop) has its own objective and context.
  • psychology
    Optimization is Memory: Optimizers (like Adam) are actually associative memories compressing gradients!
Neuroscience Inspiration

Learning from the Brain

The human brain doesn't use a single clock for updates. It uses oscillations (brain waves) at different frequencies to manage memory and cognition. Nested Learning mimics this structure.

The brain distributes memory across regions with different update rates, unlike standard LLMs which have only "frozen weights" and "immediate context".

The M3 Optimizer

Standard optimizers (Adam) act like "Goldfish" â€” they only remember the recent past gradients. The paper introduces Multi-scale Momentum Muon (M3).

M3 uses the Continuum Memory System concept inside the optimizer itself, allowing it to escape local minima by recalling long-term landscape features.

Key Terminology

Nested Learning (NL)

Viewing models as nested optimization loops rather than stacked layers.

DGD (Delta Gradient Descent)

A generalized gradient update rule that depends on the current state, unlike standard SGD.

CMS (Continuum Memory)

A memory system with a spectrum of update frequencies (Fast -> Slow).

Self-Referential

A model that generates its own learning parameters (keys, values, learning rates).
